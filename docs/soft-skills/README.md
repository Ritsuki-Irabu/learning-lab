# ソフトスキル 技術資産ドキュメント

> 面談・コミュニケーション・学習習慣など、技術以外のスキルを体系化した資産。
> 振り返りの蓄積と再現性のある改善を目的とする。

---

## 技法リファレンス

### PREP法（結論ファーストの話法）

| 要素 | 役割 | ポイント |
|---|---|---|
| Point（結論） | 最初に主張を言い切る | 「〜だと考えます」で締める |
| Reason（理由） | なぜそう言えるか | 1〜2文に絞る |
| Example（具体例） | STARで肉付け | チラ見OKで暗記不要 |
| Point（再結論） | 最後に主張を繰り返す | 「〜で貢献できます」で締める |

### STAR法（経験の構造化）

| 要素 | 役割 | 記述例 |
|---|---|---|
| Situation（状況） | 背景・環境を伝える | 案件名・チーム規模・技術スタック |
| Task（課題） | 何を解決すべきだったか | 工数・品質・証明すべきこと |
| Action（行動） | 自分が取った行動 | 技術・自走・調整の3軸で整理 |
| Result（成果） | 定量・定性の結果 | 数値化できるものは数値で |

## 面談振り返り

### 2026-06-07 グループ面談

#### 良い点

| カテゴリ | 内容 |
|---|---|
| 一貫性 | 広く浅い学習スタイルの説明と逆質問（定着の遅さへの問い）がマッチしていた |
| 準備 | 切り返しを予測して対策できていた（AIに丸投げでOK） |
| 話し方 | テキストの棒読み感を出さず、アドリブも交えて伝えられた |
| メンタル | 面談前の状態管理ができており、緊張せず言葉が出てきた |
| 積極性 | グループ面談で遠慮せず、一歩も引かずアプローチできた |
| 表現 | 話し言葉を意識して意図的に崩した表現ができた（着飾ると本心から遠のく） |

#### 反省点

| カテゴリ | 内容 | 対策 |
|---|---|---|
| 技術理解の深さ | 「自信がある」と言いつつ下調べが不十分（例：Java/Spring Boot） | 言及する技術は事前に一問一答を用意する |
| 技術への感情の言語化 | 「その技術のどこが好きか」具体的に話せなかった | 使った感想・気づきをセットで準備する |
| 表情管理 | 受けが悪いと感じたときの感情が顔に出やすい | 反応に関係なく話し切る練習をする |
| PREP定着 | 結論ファーストが咄嗟に出てこない場面があった | 日常会話でもPREP構造を意識する |

## 今後の強化方針

- [ ] 言及する技術ごとに「好きな理由・印象に残った場面」を1〜2文メモしておく
- [ ] 面談直後に振り返りを記録して改善ループを回す
- [ ] 日常会話でPREP構造を意識して、咄嗟に出るようにする
- [ ] 表情は鏡や録画で定期チェックする
