## チートシート

### robots.txt

`robots.txt` はサイトルートに置くテキストファイルで、検索エンジンなどのクローラーにクロールしてほしくない範囲を伝える。

```txt
User-agent: *
Disallow: /tmp/
Disallow: /search/
```

ただし、これはアクセス制御ではない。クローラーが従うことを期待するルールであり、悪意ある bot や仕様に従わない bot は無視できる。

### 秘密のURLを書かない

`robots.txt` は誰でも読める。

```txt
User-agent: *
Disallow: /secret-admin-panel/
```

このように書くと、隠したい URL の存在を外部に知らせるヒントになる。管理画面、未公開キャンペーン、認証が必要な資料の場所は、`robots.txt` で隠そうとしない。

### 目的別の使い分け

| 目的 | 手段 |
|---|---|
| クロール負荷を下げたい | `robots.txt` |
| 検索結果に出したくない | `noindex` / 削除 |
| 内容を見せたくない | 認証・認可・Basic認証・IP制限 |
| 管理画面を守りたい | 認証・認可・2FA・IP制限 |

## 技術的課題と改善方針

| 課題 | 問題点 | 改善方針 |
|---|---|---|
| robots.txt を秘密保持に使う | 誰でも読めるため秘密URLの露出につながる | 秘密保持は認証・認可で行う |
| クロール制御と検索除外の混同 | robots.txt だけでは検索結果制御として不十分な場合がある | noindex など目的に合う手段を使う |
