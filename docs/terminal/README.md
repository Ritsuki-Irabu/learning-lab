## チートシート

### vi / vim

```bash
vi index.php
```

`index.php` を Vim で開く。存在しない場合でも、保存時に新規作成される。

| 操作 | 意味 |
|---|---|
| `i` | insert mode に入り、編集できるようにする |
| `Esc` | normal mode に戻る |
| `:wq` | 保存して終了する |
| `:q!` | 保存せず強制終了する |

```text
vi index.php
i
編集する
Esc
:wq
Enter
```

> ⚠️ 保存終了は `:qw` ではなく `:wq`。`write` の `w`、`quit` の `q` と覚える。

### 隠しファイル表示

```bash
ls -la
```

| オプション | 意味 |
|---|---|
| `-l` | 詳細表示 |
| `-a` | `.env` などの隠しファイルも表示 |

Laravel の `.env`、Git の `.gitignore`、`.git` など、ドットで始まるファイルやディレクトリを確認するときに使う。

### Tab補完

```bash
cd lar<Tab>
```

フォルダ名やファイル名は、先頭の数文字 + `Tab` で補完できる。候補が複数ある場合は、追加で `Tab` を押すと候補一覧が出ることがある。

## 技術的課題と改善方針

| 課題 | 問題点 | 改善方針 |
|---|---|---|
| Vimで文字入力できない | 起動直後がnormal modeだと知らないと詰まる | `i` → 編集 → `Esc` → `:wq` の流れで覚える |
| `.env` が見えない | 通常の `ls` では隠しファイルが表示されない | `ls -la` を使う |
| パス入力が遅い | ファイル名を毎回手入力してミスが増える | Tab補完を使う |
